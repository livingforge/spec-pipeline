# 脅威モデル — docsummary

想定する信頼境界と、各層の防御、そして**それを検証するテスト**を対応づける文書。
関連: [GOVERNANCE.md](GOVERNANCE.md) / [dependencies.md](dependencies.md)。

## スコープと前提

- docsummary は**登録済み文書を外部 LLM で要約する**ツール。要約のため文書本文を
  外部プロバイダへ送信する（データ egress を伴う）点が他スキルと異なる主眼。
- 想定利用者は、自組織の資料を自分の LLM 契約で要約するエンジニア／エージェント。
- 主眼は「**秘密情報（API キー）を漏らさないこと**」「**外部送信を利用者の承知の下で
  行うこと**」「**要約対象・結果の取り違えや黙った失敗を避けること**」。

## 信頼境界（trust boundaries）

| # | 境界 | 内→外 / 外→内 | 主なリスク |
|---|---|---|---|
| B1 | 設定・秘密情報（`.env` / env） | 外→内（キーの取り込み） | API キーのハードコード・ログ/出力への漏洩・コミット |
| B2 | docsummary → LLM プロバイダ | 内→外（文書本文の送信） | 機微文書の意図しない外部送信・プロバイダ選定の誤り |
| B3 | 入力 `library.json` → 要約 | 外→内（兄弟スキルの成果物） | 未登録/欠損文書の取り違え・黙った失敗 |

## 脅威 → 防御層 → 検証テスト

| ID | 脅威 | 信頼境界 | 防御層 | 検証テスト |
|---|---|---|---|---|
| T1 | API キーがコード・要約ストア・ログ・エラー出力に載って漏洩する、または誤ってコミットされる | B1 | **D1**: 秘密はコードに持たず `.env`・環境変数経由でのみ受け取り、**値は表示・保存しない**（`config` はキーの有無/参照先のみ提示）。`.env` はコミット対象外 | `tests/test_docsummary.py`（キー値を出力・保存しないこと／設定解決） |
| T2 | 機微文書の本文が利用者の承知なく外部 LLM へ送信される | B2 | **D2**: 送信先は利用者が明示設定したプロバイダのみ（`config` で選択）。既定でどのプロバイダにも自動送信しない。プロバイダは `urllib` で明示的に呼び、送信経路が設定に一致する。送信可否・機微判定は運用側の責務として明記 | `tests/test_docsummary.py`（設定したプロバイダのみ呼ぶ／未設定時は呼ばない） |
| T3 | API キー未設定・接続失敗を「要約なし」と黙って扱い、欠落に気づけない | B1/B2 | **D3**: キー未設定・接続失敗は fail-closed で明確にエラー提示（黙って成功扱いにしない） | `tests/test_docsummary.py`（未設定/失敗時のエラー） |
| T4 | 未登録・欠損した文書を取り違えて要約する／壊れた `library.json` で全体が落ちる | B3 | **D4**: 対象は docagent 索引（`library.json`）から解決し、未登録は要約対象にしない。鮮度は内容ハッシュ + 仕様ハッシュで判定し `--pending` で未要約・陳腐化のみ対象化 | `tests/test_docsummary.py`（対象選択・鮮度判定） |

> 上表のテスト名は同梱テストの意図を示す（実ファイルは docsummary 用テスト
> `tests/test_docsummary.py` に対応）。防御を変えたら対応テストも併せて更新する。
