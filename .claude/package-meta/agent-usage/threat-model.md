# 脅威モデル — agent-usage

想定する信頼境界と、各層の防御、そして**それを検証するテスト**を対応づける文書。
関連: [GOVERNANCE.md](GOVERNANCE.md) / [dependencies.md](dependencies.md)。

## スコープと前提

- agent-usage は**ローカルのエージェントログを読み取り、集計を出力するだけ**の
  ツールであり、ネットワークサービスとして公開されない。外部依存もネットワーク
  アクセスも持たない（オフライン・読み取り専用）。
- 想定利用者は、自分（または自組織）の Claude Code / GitHub Copilot のログを集計する
  エンジニア／エージェント。
- したがって主眼は「**入力ログを書き換えない/壊さないこと**」「**壊れた/欠損した
  ログを黙って誤集計しないこと**」「**生成物 `report.html` に機微情報を不用意に
  埋め込まないこと**」であり、ネットワーク攻撃者対策ではない。

## 信頼境界（trust boundaries）

| # | 境界 | 内→外 / 外→内 | 主なリスク |
|---|---|---|---|
| B1 | セッションログ → コレクタ | 外→内（利用環境が書いたログの読み取り） | 破損/欠損 JSONL による例外・誤集計 |
| B2 | 集計 → `report.html` / `summary.json` | 内→外（成果物の出力） | 機微情報（プロンプト本文・パス）の意図しない埋め込み・共有 |
| B3 | 単価表 `pricing.json` | 外→内（利用者が編集可能なデータ） | 未検証単価による誤ったコスト表示 |

## 脅威 → 防御層 → 検証テスト

| ID | 脅威 | 信頼境界 | 防御層 | 検証テスト |
|---|---|---|---|---|
| T1 | 入力ログを解析中に上書き・破壊してしまう | B1 | **D1**: ログは**読み取り専用**で開き、集計結果は `--out` 配下にのみ書き出す（入力ディレクトリを書き換えない） | `tests/test_agent_usage.py`（入力不変・出力先分離） |
| T2 | 破損/途中で切れた JSONL・欠損フィールドで集計全体が落ちる、または黙って一部を取りこぼす | B1 | **D2**: 行単位で堅牢にパースし、壊れた行は捨てずに degraded/skip として扱い件数を集計に残す（silent degradation を避ける） | `tests/test_agent_usage.py`（不正 JSON 行・usage 欠落・空セッション） |
| T3 | サブエージェント（sidechain / child session）の内部消費を集計から落とし、過小評価する | B1 | **D3**: `subagents/agent-*.jsonl` と `child_session_ref` の子ログを読み、親呼び出しへひも付けて総計へ算入。古いログで内部が取れない場合は HTML に**過小評価の注意**を明示 | `tests/test_agent_usage.py`（サブエージェント集入・注意表示） |
| T4 | `report.html` にプロンプト本文・パス等の機微情報が埋め込まれ、共有時に漏洩する | B2 | **D4**: HTML はタイムライン等でツール名・件数・時間・コストなどの**メタ情報**を中心に構成し、本文全文は既定で載せない。埋め込むテキストは HTML エスケープして注入を防ぐ。出力の取り扱い（共有可否）は運用側の責務であることを前提化 | `tests/test_agent_usage.py`（HTML エスケープ・自己完結性） |
| T5 | 未検証・古い単価で誤ったコストを「正しい」ように提示する | B3 | **D5**: `pricing.json` の `verified: false` の間は HTML 上に警告を出し、表に無いモデルはコストを unknown（`*`）として区別（黙って 0 円・概算にしない） | `tests/test_agent_usage.py`（未検証警告・未知モデルの unknown 表示） |

> 上表の検証テスト名は同梱テストの意図を示す（実ファイル名は `tests/` 配下の
> agent-usage 用テストに対応）。防御を追加・変更したら、対応するテストを必ず併せて
> 更新し、テストの無い防御を残さない（[GOVERNANCE.md](GOVERNANCE.md) の棚卸し周期）。
